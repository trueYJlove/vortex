# 分组头轻强调设计

## 目标

让侧栏中的一级分组头（如“任务计划”“会话”“已修改文件”）在不增加明显视觉负担的前提下更容易被快速识别。这个调整只改变层级感和可扫读性，不改变信息结构，也不增加新的交互复杂度。

## 范围

本设计只覆盖以下区域的分组头样式：

- 任务计划分组头
- 会话分组头
- 已修改文件分组头
- 其他同级侧栏分组头，如果它们当前复用相同标题结构

不包含：

- 分组内容项样式
- 分组排序
- 分组折叠逻辑
- 新增筛选、搜索或计数规则

## 设计原则

1. **轻强调，不抢内容**
   - 分组头应比内容项更像“区域标题”，但不能像独立卡片。
   - 视觉变化控制在字重、间距、边界感三个维度内。

2. **保持一致性**
   - 所有一级分组头使用同一套视觉规则。
   - 仅对需要状态提示的分组保留附加文案或数量信息。

3. **保持现有密度**
   - 不显著增加侧栏整体高度。
   - 不引入额外装饰性元素，避免侧栏变碎。

## 方案定义

### 标题层级

- 标题文字使用更明确的字重，让用户第一眼知道这是分组边界。
- 标题与分组内容之间保留轻微间距。
- 分组头与上下区域之间增加更稳定的分隔感。

### 辅助信息

- 对于“已修改文件”这类状态分组，可在标题后保留轻量提示，例如“需要 Git”。
- 提示文案应保持次级语义，不压过主标题。
- 提示只承担说明作用，不承担主要视觉强调。

### 交互保持不变

- 分组头仍然沿用现有折叠/展开交互。
- 点击区域不缩小，不改变当前可点击范围。
- 现有内容项交互保持不变。

## 组件边界

### 适用组件

- `ArtifactRail` 中的相关分组头
- `ConversationList` 中的标题区域
- `GitChangesPanel` 中的分组标题区域
- 其他复用同类标题结构的侧栏区域

### 建议做法

如果多个分组头当前写法不同，优先抽象成统一的标题样式组合，而不是为每个组件单独调参。这样可以：

- 减少视觉不一致
- 避免后续继续分叉
- 让同级分组保持相同的语义强度

## 验收标准

- 用户能更快区分分组头和内容项。
- 侧栏不会明显变高或变重。
- “任务计划 / 会话 / 已修改文件”三个标题在视觉上有统一的层级感。
- 现有点击、折叠、拖拽行为不变。
- 在窄屏下仍然保持整洁，不出现拥挤或过度装饰。

## 测试关注点

- 视觉回归：分组头是否比内容项更醒目，但不喧宾夺主。
- 响应式：在 `< 640px` 宽度下是否仍然清晰。
- 一致性：各分组头是否保持统一的字重、间距和提示层级。
- 交互：折叠/展开与点击区域是否未受影响。
